# 🚢 Система учета грузов морского терминала

## 📌 Описание

Веб-приложение для автоматизации учета и управления грузовыми операциями морского порта. Система обеспечивает полный контроль над движением грузов через различные виды транспорта (суда, железнодорожные вагоны, грузовики), управление складскими помещениями и причалами.

## 🎯 Основной функционал

### 1. **Управление судами и судозаходами (Vessel Management)**
- Регистрация морских судов с полными характеристиками (название, IMO, тип, дедвейт, вместимость)
- Учет судозаходов (Vessel Calls) с отслеживанием:
  - ETA/ETD (плановое прибытие/отправление)
  - ATA/ATD (фактическое прибытие/отправление)
  - Назначение причала
  - Статус операций (EXPECTED, BERTHED, LOADING, UNLOADING, DEPARTED)

### 2. **Управление причалами (Berth Management)**
- Регистрация причалов с характеристиками:
  - Длина, глубина, грузоподъемность крана
  - Тип груза (контейнеры, навалочные, наливные, универсальные)
  - Статус доступности (AVAILABLE, OCCUPIED, MAINTENANCE)
- Планирование и контроль загрузки причалов

### 3. **Управление контейнерами (Container Management)**
- Учет контейнеров по стандарту ISO 6346
- Отслеживание перемещений между:
  - Судами
  - Вагонами
  - Грузовиками
  - Складами
- Контроль статусов (AT_BERTH, AT_WAREHOUSE, ON_WAGON, ON_TRUCK, IN_TRANSIT)
- Типы контейнеров (20', 40', 40' HC, рефконтейнеры, танк-контейнеры)

### 4. **Управление железнодорожными вагонами (Wagon Management)**
- Регистрация вагонов с параметрами:
  - Номер, тип (крытый, платформа, цистерна, полувагон)
  - Грузоподъемность, объем
  - Статус (LOADED, EMPTY, LOADING, UNLOADING)
- Планирование погрузочно-разгрузочных работ

### 5. **Управление грузовиками (Truck Management)**
- Регистрация грузовиков (номер, тип, грузоподъемность)
- Учет визитов грузовиков (Truck Visits):
  - Время въезда/выезда
  - Привязка к контейнерам
  - Статус (REGISTERED, AT_GATE, LOADING, UNLOADING, DEPARTED)

### 6. **Управление складами (Warehouse Management)**
- Регистрация складских площадей:
  - Типы (открытая площадка, крытый склад, рефрижераторный, таможенный, площадка для пустых контейнеров)
  - Вместимость (TEU), занятость
  - Статус (ACTIVE, INACTIVE, FULL, MAINTENANCE)
- Отслеживание размещения контейнеров

### 7. **Дашборд и аналитика**
- Статистика в реальном времени:
  - Активные судозаходы
  - Загрузка причалов
  - Количество контейнеров по статусам
  - Занятость складов
  - Активные вагоны и грузовики
- Визуализация ключевых показателей эффективности (KPI)

## 🚀 Реализация

### Backend (REST API)

#### Endpoints структура:
```
/api/vessels              - CRUD операции с судами
/api/vessel-calls         - Управление судозаходами
/api/berths               - Управление причалами
/api/containers           - Учет контейнеров
/api/wagons               - Управление вагонами
/api/trucks               - Регистрация грузовиков
/api/truck-visits         - Визиты грузовиков
/api/warehouses           - Управление складами
/api/stats                - Статистика и аналитика
```

#### Модель данных (Prisma Schema):
- **Vessel**: суда с характеристиками
- **VesselCall**: судозаходы с привязкой к судам и причалам
- **Berth**: причалы с параметрами
- **Container**: контейнеры с отношениями к различным транспортным средствам
- **Wagon**: железнодорожные вагоны
- **Truck**: грузовики
- **TruckVisit**: визиты грузовиков
- **Warehouse**: складские помещения

#### Статусные модели:
- Суда: EXPECTED, BERTHED, LOADING, UNLOADING, DEPARTED
- Причалы: AVAILABLE, OCCUPIED, MAINTENANCE
- Контейнеры: AT_BERTH, AT_WAREHOUSE, ON_WAGON, ON_TRUCK, IN_TRANSIT
- Вагоны: LOADED, EMPTY, LOADING, UNLOADING
- Визиты грузовиков: REGISTERED, AT_GATE, LOADING, UNLOADING, DEPARTED
- Склады: ACTIVE, INACTIVE, FULL, MAINTENANCE
