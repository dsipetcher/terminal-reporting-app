# 🚀 Упрощенное развертывание Terminal Operating System

## 🎯 3 способа развертывания

---

## ✨ Способ 1: SQLite (Самый простой - БЕЗ PostgreSQL)

### Преимущества:
- ✅ Не нужен PostgreSQL
- ✅ База данных создается автоматически
- ✅ Один файл `dev.db`
- ✅ Идеально для разработки и тестирования

### Шаги:

**1. Обновить schema.prisma (уже сделано):**
```prisma
datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}
```

**2. Запустить миграции:**
```bash
cd backend
npm install
npx prisma migrate dev --name init
npx prisma generate
npm run dev
```

**3. В отдельном терминале - Frontend:**
```bash
cd frontend
npm install
npm run dev
```

✅ **Готово!** База данных SQLite создана автоматически в `backend/dev.db`

## 🐳 Способ 2: Docker Compose (Полная изоляция)

### Преимущества:
- ✅ Все в контейнерах
- ✅ PostgreSQL включен
- ✅ Одна команда для запуска
- ✅ Изолированная среда

### Требования:
- Docker Desktop установлен

### Шаги:

**1. Запустить все сервисы:**
```bash
docker-compose up --build
```

**2. Открыть в браузере:**
- Frontend: http://localhost:5173
- Backend: http://localhost:3001
- PostgreSQL: localhost:5432

**3. Остановить:**
```bash
docker-compose down
```

**4. Удалить все данные:**
```bash
docker-compose down -v
```

## 🌐 Способ 3: GitHub Pages + Бесплатный хостинг

### Архитектура:
- **Frontend** → GitHub Pages (статика)
- **Backend** → Railway.app / Render.com (бесплатно)
- **База данных** → В комплекте с Railway/Render

### Шаги:

#### A. Развертывание Backend на Railway

**1. Создать аккаунт на [Railway.app](https://railway.app)**

**2. Установить Railway CLI:**
```bash
npm install -g @railway/cli
```

**3. Войти и развернуть:**
```bash
cd backend
railway login
railway init
railway add postgresql
railway up
```

**4. Получить URL backend:**
```bash
railway domain
```
Пример: `https://your-app.railway.app`

#### B. Развертывание Frontend на GitHub Pages

**1. Обновить API URL в frontend:**

Создать файл `frontend/.env.production`:
```env
VITE_API_URL=https://your-app.railway.app
```

**2. Обновить `frontend/src/api/index.ts`:**
```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001/api';
```

**3. Собрать frontend:**
```bash
cd frontend
npm run build
```

**4. Развернуть на GitHub Pages:**

Создать файл `.github/workflows/deploy.yml`:
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install and Build
        run: |
          cd frontend
          npm install
          npm run build
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./frontend/dist
```

**5. Настроить GitHub Pages:**
- Перейти в Settings → Pages
- Source: Deploy from a branch
- Branch: gh-pages

✅ **Готово!** Frontend на `https://username.github.io/terminal-reporting-app`

## 🔧 Скрипты для быстрого запуска

### Создать `start.bat` для Windows (SQLite):

```batch
@echo off
echo Starting Terminal Operating System...

echo.
echo [1/3] Starting Backend...
cd backend
start cmd /k "npm run dev"

timeout /t 3

echo.
echo [2/3] Starting Frontend...
cd ../frontend
start cmd /k "npm run dev"

echo.
echo [3/3] Opening browser...
timeout /t 5
start http://localhost:5173

echo.
echo ===================================
echo   Terminal Operating System
echo ===================================
echo   Frontend: http://localhost:5173
echo   Backend:  http://localhost:3001
echo ===================================
```

### Создать `start.sh` для Linux/Mac:

```bash
#!/bin/bash
echo "Starting Terminal Operating System..."

echo "[1/3] Starting Backend..."
cd backend
npm run dev &
BACKEND_PID=$!

sleep 3

echo "[2/3] Starting Frontend..."
cd ../frontend
npm run dev &
FRONTEND_PID=$!

sleep 5

echo "[3/3] Opening browser..."
open http://localhost:5173 || xdg-open http://localhost:5173

echo "==================================="
echo "  Terminal Operating System"
echo "==================================="
echo "  Frontend: http://localhost:5173"
echo "  Backend:  http://localhost:3001"
echo "==================================="
echo ""
echo "Press Ctrl+C to stop all services"

wait $BACKEND_PID $FRONTEND_PID
```

## 🆘 Решение проблем

### Проблема: Prisma не находит DATABASE_URL

**Решение для SQLite:**
Не нужна переменная окружения! База создается автоматически.

**Решение для PostgreSQL:**
```bash
# Windows PowerShell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/terminal_db"
npx prisma migrate dev

# Linux/Mac
export DATABASE_URL="postgresql://postgres:postgres@localhost:5432/terminal_db"
npx prisma migrate dev
```

### Проблема: Миграции не применяются

**Решение:**
```bash
# Сбросить БД и начать заново
npx prisma migrate reset --force

# Или использовать db push
npx prisma db push --accept-data-loss
```
