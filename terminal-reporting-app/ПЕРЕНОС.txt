ИЛС — запуск одной командой (без установки Node.js)
==================================================

БЫСТРЫЙ СТАРТ
  Дважды щёлкните:  run.bat

  Откроется http://localhost:3001
  Логин: admin
  Пароль: admin

КАК ЭТО РАБОТАЕТ
  run.bat автоматически:
    • использует portable Node.js из runtime/node в корне репозитория
    • или скачивает его один раз (нужен интернет): install-node.bat
    • или использует системный Node.js, если он установлен
    • ставит зависимости, если их ещё нет
    • создаёт базу SQLite с демо-данными, если её нет
    • собирает интерфейс, если ещё не собран
    • запускает ОДИН сервер: API + сайт на порту 3001

ПОДГОТОВКА ФЛЕШКИ (на вашем ПК, один раз)
  1. prepare-portable.bat  — всё скачает и соберёт (~5–10 мин, нужен интернет)

  В PowerShell обязательно с точкой и слэшем:
    .\prepare-portable.bat
  (из корня репозитория или из папки terminal-reporting-app)

  Перед подготовкой закройте run.bat / start.bat или выполните stop.bat
  2. Скопируйте всю папку terminal-reporting-app на флешку

  На другом компьютере:
    • Node.js ставить НЕ нужно
    • Интернет НЕ нужен (если скопировали runtime/node и node_modules)
    • Просто run.bat

ЧТО КОПИРОВАТЬ НА ФЛЕШКУ (полный offline-набор)
  Весь репозиторий или как минимум:
    run.bat
    install-node.bat
    prepare-portable.bat
    repair.bat
    runtime/node/          ← portable Node.js (корень репозитория)
    terminal-reporting-app/
      backend/prisma/dev.db  ← демо-БД (есть в git)
      frontend/dist/         ← собранный интерфейс

ДЕМО-ДАННЫЕ (13 партий, составы, документы)
  Скрипт seed.ts лежит в репозитории.
  run.bat при каждом запуске проверяет БД и при необходимости
  заполняет её: npm run db:ensure-data

  Если данных нет — удалите backend/prisma/dev.db и запустите run.bat
  или выполните: npm run db:ensure-data

ЕСЛИ npm install ПАДАЕТ (EPERM, EISDIR, Prisma)
  1. git pull origin main
  2. Закройте run.bat (или stop.bat)
  3. Запустите repair.bat из корня репозитория
  4. Затем run.bat

  repair.bat удалит node_modules и поставит зависимости заново.
  В PowerShell: .\repair.bat
  Не копируйте node_modules с другого ПК

ОСТАНОВКА
  stop.bat  или закройте окно консоли

РАЗРАБОТКА (горячая перезагрузка)
  start.bat — отдельно backend + frontend (как раньше)
