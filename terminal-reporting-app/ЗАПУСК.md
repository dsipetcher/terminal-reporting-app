# 🚢 Terminal Operating System - Руководство по запуску

## Описание проекта

**Terminal Operating System (TOS)** - полнофункциональная система управления операциями морского порта.

### Реализованный функционал

✅ **Морской фронт:**
- Управление судами и судозаходами
- Управление причалами
- Отслеживание статусов

✅ **Грузы:**
- Учет контейнеров (стандарт ISO 6346)
- Управление ж/д вагонами
- Адресное хранение

✅ **Транспорт:**
- Автотранспорт с тайм-слотами
- Контроль въезда/выезда

✅ **Инфраструктура:**
- Управление складами
- Расчет загрузки

✅ **Dashboard:**
- Оперативная статистика
- Активные судозаходы
- Быстрые действия

## 🚀 Быстрый запуск

### Предварительные требования

- Node.js >= 18.x
- PostgreSQL >= 14.x
- npm или yarn

### 1. Настройка базы данных

```bash
# Создать базу данных PostgreSQL
createdb terminal_db

# Создать файл .env в папке backend
cd backend
echo "DATABASE_URL=postgresql://user:password@localhost:5432/terminal_db" > .env
```

### 2. Установка и запуск Backend

```bash
cd backend

# Установить зависимости
npm install

# Сгенерировать Prisma Client
npm run prisma:generate

# Применить миграции
npm run prisma:migrate

# Запустить сервер разработки
npm run dev
```

Backend запустится на `http://localhost:3001`

### 3. Установка и запуск Frontend

```bash
cd frontend

# Установить зависимости
npm install

# Запустить сервер разработки
npm run dev
```

Frontend запустится на `http://localhost:5173`

## 📡 API Endpoints

### Vessels
- `GET /api/vessels` - Получить все суда
- `GET /api/vessels/:id` - Получить судно по ID
- `POST /api/vessels` - Создать судно
- `PUT /api/vessels/:id` - Обновить судно
- `DELETE /api/vessels/:id` - Удалить судно

### Vessel Calls
- `GET /api/vessel-calls` - Получить все судозаходы
- `GET /api/vessel-calls/:id` - Получить судозаход
- `POST /api/vessel-calls` - Создать судозаход
- `PUT /api/vessel-calls/:id` - Обновить судозаход
- `PATCH /api/vessel-calls/:id/status` - Обновить статус
- `DELETE /api/vessel-calls/:id` - Удалить судозаход

### Berths
- `GET /api/berths` - Получить все причалы
- `POST /api/berths` - Создать причал
- `PUT /api/berths/:id` - Обновить причал

### Containers
- `GET /api/containers` - Получить все контейнеры
- `GET /api/containers/number/:containerNumber` - Поиск по номеру
- `POST /api/containers` - Создать контейнер
- `PATCH /api/containers/:id/move` - Переместить контейнер

### Trucks & Visits
- `GET /api/trucks` - Получить все грузовики
- `GET /api/truck-visits` - Получить все визиты
- `PATCH /api/truck-visits/:id/check-in` - Отметить въезд
- `PATCH /api/truck-visits/:id/check-out` - Отметить выезд

### Wagons
- `GET /api/wagons` - Получить все вагоны
- `POST /api/wagons` - Создать вагон
- `PATCH /api/wagons/:id/status` - Обновить статус

### Warehouses
- `GET /api/warehouses` - Получить все склады
- `POST /api/warehouses` - Создать склад

### Dashboard
- `GET /api/dashboard/stats` - Получить статистику
