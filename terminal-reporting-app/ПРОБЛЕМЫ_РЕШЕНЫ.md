## 🔧 Технические изменения

### База данных

**Было:** PostgreSQL с внешней установкой
```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

**Стало:** SQLite с автоматическим созданием
```prisma
datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}
```

### Типы данных

**Было:** Enum (не поддерживается в SQLite)
```prisma
enum VesselType {
  CONTAINER
  BULK_CARRIER
  TANKER
}
```

**Стало:** String с комментариями
```prisma
vesselType String // CONTAINER, BULK_CARRIER, TANKER, RORO, GENERAL_CARGO, OTHER
```

### API конфигурация

**Добавлено:**
```typescript
// frontend/src/api/index.ts
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:3001/api';
```

Теперь API URL настраивается через переменные окружения!

## 📊 Сравнение подходов

| | PostgreSQL (старый) | SQLite (новый) |
|---|---|---|
| **Установка** | Требуется установка | Не требуется |
| **Настройка** | .env + DATABASE_URL | Автоматически |
| **Миграции** | Ошибки с env vars | Работает с первого раза |
| **База данных** | Внешний сервер | Один файл `dev.db` |
| **Для разработки** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **Для продакшена** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

## ⚠️ Важные примечания

### Когда использовать SQLite:
✅ Разработка  
✅ Тестирование  
✅ Демонстрация  
✅ Локальный запуск  
✅ Обучение  

### Когда использовать PostgreSQL (Docker):
✅ Продакшен  
✅ Много пользователей  
✅ Критичные данные  
✅ Масштабирование  

### Переключение на PostgreSQL

Если в будущем понадобится PostgreSQL, просто измените `schema.prisma`:

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

И запустите Docker:
```bash
docker-compose up
```
